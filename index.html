01 — Origin & Philosophy

Why Fluent?

Every programming language since FORTRAN has been a compromise — a negotiation between human thought and machine execution, dressed in symbols that humans must learn to speak. Fluent abandons this compromise entirely.

Fluent is written as you would instruct a deeply knowledgeable colleague: in complete, grammatical, English-like sentences. The runtime translates your intent into deterministic execution. There are no semicolons. No curly braces. No type annotations. Only expression.

At the same time, Fluent is built around a single, radical idea: AI models are first-class citizens. Calling GPT-4o, Claude 3.7, Gemini 2.5, Mistral, Llama, Cohere — or any registered model — requires no SDK, no HTTP boilerplate, no JSON marshalling. A single natural-language sentence invokes the model, receives the result, and continues the flow.

Human First

Programs read like paragraphs. Syntax emerges from grammar, not tokenomics. If you can describe it, you can program it.

AI Native

Model invocation is a built-in primitive, not a library call. The type system understands "a response from a language model."

Provider Agnostic

One unified interface covers every model from every provider. Switch from Claude to Gemini by changing one word.

Design Principle: A Fluent program should be readable — and partially understandable — by someone who has never seen code before.
02 — Language Syntax

Syntax Reference

Fluent has seven syntactic constructs. Each maps directly to English grammatical patterns. There is no formal grammar beyond these seven forms.

Declaration

Let [name] be [value or expression].

Assignment

Set [name] to [value or expression].

Model Invocation

Ask [model] to [instruction] and call the result [name].

Conditional

If [condition], then [action]. Otherwise, [action].

Iteration

For each [item] in [collection], [action].

Pipeline

Pass [value] through [step1], then through [step2].

Definition

To [verb phrase]: [body]. End of [name].

Error Handling

Try to [action]. If that fails, [fallback].

Comments & Annotations

comments.fl Fluent 
-- This is a single-line comment.

-- Annotations attach metadata to any declaration.
@model(temperature: 0.7, max_tokens: 500)
Ask claude to "summarize the quarterly report" and call the result summary.

Data Types

Fluent infers types from context. You may optionally declare types as English phrases.

types.fl Fluent 
-- Scalar types
Let user_name be "Harshal".               -- text
Let score be 42.                           -- number
Let is_active be true.                     -- boolean
Let today be the current date.              -- date (inferred)

-- Collections
Let languages be a list containing "Python", "Rust", and "Go".
Let profile be a record with name "Harshal", age 30, and city "Mumbai".

-- AI-native types
Let reply be a model response.              -- typed placeholder
Let embedding be a vector of 1536 numbers. -- embedding vector
Let frame be an image.                      -- image type
03 — Model Binding

Invoking AI Models

Fluent treats model invocations as typed expressions, not side effects. Every "Ask" statement is pure: given the same inputs and parameters, it returns a deterministic result (seeded by temperature).

Basic Invocation

invoke.fl Fluent 
-- Minimal: ask any known model alias
Ask claude to "explain quantum entanglement simply" and call the result explanation.

-- With a variable prompt
Let topic be "black holes".
Ask gpt to "write a haiku about" topic and call the result haiku.

-- Fully qualified model reference
Ask anthropic/claude-opus-4 to "analyze this legal contract for risks"
    using document contract_text
    and call the result risk_analysis.

Parameters & Configuration

-- Inline parameter annotation
@model(temperature: 0.2, max_tokens: 1000, format: json)

-- Multimodal: text + image

05 — Control Flow

Control & Logic

Conditionals

conditions.fl Fluent 
-- Simple conditional
If sentiment is "negative",
    then ask claude to "draft an empathetic apology response" and call the result reply.
Otherwise, let reply be "Thank you for your feedback!".

-- Multi-branch (when / otherwise when / finally)
When risk_score is greater than 0.9, then output "Critical risk — halt process.".
Otherwise when risk_score is greater than 0.6, then output "Elevated risk — flag for review.".
Otherwise when risk_score is greater than 0.3, then output "Low risk — proceed with caution.".
Finally, output "Risk within acceptable bounds.".

Iteration

iteration.fl Fluent 
-- For-each loop
For each article in articles:
    Ask claude to "summarize this article in one sentence"
        using text article.body
        and call the result article.summary.
End loop.

-- Parallel iteration (concurrent model calls)
In parallel, for each document in document_batch:
    @model(temperature: 0)
    Ask gemini to "extract key facts as a JSON list"
        using text document
        and call the result document.facts.
End parallel loop.

-- While loop
While attempts is less than 3 and result is empty:
    Ask gpt to "generate a valid JSON response for the schema"
        using schema expected_schema
        and call the result result.
    Set attempts to attempts plus 1.
End while.

Error Handling

errors.fl Fluent 
-- Try / fallback
Try to ask openai/gpt-4o to "process this request"
    using text user_input
    and call the result response.
If that fails, try anthropic/claude-sonnet-4-6 with the same instruction.
If that also fails, set response to "Service temporarily unavailable.".

-- Retry with backoff
@retry(times: 3, backoff: exponential)
Ask claude to "translate the document to French"
    using document english_doc
    and call the result french_doc.
06 — Multi-Model Pipelines

AI Pipelines

Fluent's most powerful construct is the multi-model pipeline — a linear or branching chain of AI calls where the output of each step feeds the next. This is idiomatic Fluent, not a feature bolted on.

Input
raw_text
→
Step 1
Extract (claude)
→
Step 2
Translate (gemini)
→
Step 3
Polish (gpt)
→
Output
final_text
pipeline.fl Fluent 
-- Multi-model document processing pipeline

Let raw_document be read file "contract.pdf".

-- Step 1: Extract key clauses (specialized model)
@model(temperature: 0, format: json)
Ask anthropic/claude-opus-4 to "extract all obligation clauses as a structured list"
    using document raw_document
    and call the result clauses.

-- Step 2: Risk-score each clause (fast, cheap model)
In parallel, for each clause in clauses:
    @model(temperature: 0.1)
    Ask groq/llama-4-scout-instant to "assign a legal risk score from 0 to 1 for this clause"
        using text clause
        and call the result clause.risk.
End parallel loop.

-- Step 3: Filter high-risk clauses
Let high_risk_clauses be all items from clauses where risk is greater than 0.7.

-- Step 4: Write executive summary (premium model)
@model(temperature: 0.3, max_tokens: 800)
Ask openai/gpt-4o to "write an executive summary of legal risks for a non-lawyer audience"
    using data high_risk_clauses
    and call the result summary.

Output summary.

Mixture of Experts Pattern

mixture.fl Fluent 
-- Ask three models; synthesize with a fourth

Let question be "What will be the impact of AGI on global employment?".

In parallel:
    Ask claude to "give your perspective on" question and call the result opinion_a.
    Ask gpt to "give your perspective on" question and call the result opinion_b.
    Ask gemini to "give your perspective on" question and call the result opinion_c.
End parallel.

@model(temperature: 0.2)
Ask mistral/mistral-large-3 to "synthesize these three expert opinions into one balanced consensus view"
    using data a record with A opinion_a, B opinion_b, C opinion_c
    and call the result consensus.

Output consensus.

Example 1 — AI Customer Support Agent

support-agent.fl Fluent · ~30 lines 
-- Intelligent multi-model customer support triage system

Define model triage_bot as groq/llama-4-scout-instant
    with temperature 0,
    system "Classify tickets only. Respond with one word: billing, technical, general, or escalate.".

Define model responder as anthropic/claude-sonnet-4-6
    with temperature 0.4,
    system "You are a helpful, empathetic support agent for Saviee app. Be concise.".

To handle a support ticket (ticket):
    -- Step 1: Fast, cheap classification
    Ask triage_bot to "classify this ticket"
        using text ticket.message
        and call the result category.

    -- Step 2: Route based on category
    When category is "escalate",
        then notify human agent with ticket and output "Escalated to human agent.".

    -- Step 3: Search knowledge base for relevant info
    Search knowledge base support_docs
        for ticket.message
        and call the result relevant_articles.

    -- Step 4: Generate grounded response
    @model(max_tokens: 300)
    Ask responder to "write a helpful reply to the customer's" category "issue"
        using context relevant_articles, ticket ticket.message
        and call the result reply.

    Output reply.
End of handle a support ticket.

research.fl Fluent · ~25 lines

09 — Functions & Modules

Functions & Modules

Fluent programs are composed of named procedures — defined in natural language, invoked in natural language. Modules are plain .fl files; importing them reads exactly like English.

Defining Functions

functions.fl Fluent 
-- Define a reusable procedure
To summarize an article (article_text) in (language):
    @model(temperature: 0.3, max_tokens: 200)
    Ask claude to "summarize this article in three sentences"
        using text article_text
        and call the result raw_summary.

    If language is not "English",
        then ask gemini to "translate this to" language
            using text raw_summary
            and call the result raw_summary.

    Return raw_summary.
End of summarize an article.

-- Call it
Let brief be the result of summarize an article news_article in "Hindi".
Output brief.

Recursive Functions

recursion.fl Fluent 
-- Recursive document decomposition
To split and process (document) until each chunk is under 500 words:
    If the word count of document is at most 500,
        then ask claude to "extract key facts from this chunk"
            using text document
            and return the result.
    Otherwise:
        Let first_half be the first half of document.
        Let second_half be the second half of document.
        Let facts_a be the result of split and process first_half.
        Let facts_b be the result of split and process second_half.
        Return facts_a combined with facts_b.
End of split and process.

Modules & Imports

10 — Agentic Loops

Autonomous Agents

Fluent has first-class support for agentic loops — programs where a model reasons about what action to take next, executes it, observes the result, and decides whether the goal has been achieved. This is expressed, naturally, in plain sentences.

The Agent Loop

agent.fl Fluent 
-- A goal-directed agent with tools
@tools(web_search, read_file, write_file, run_python, send_email)
@agent(max_steps: 20, model: anthropic/claude-opus-4)
Let goal be "Analyze the sales data in sales_2026.csv, find the top 3 underperforming regions,
            research industry benchmarks online, and email a concise strategy report to team@company.com".

Run agent with goal goal and call the outcome agent_result.

If agent_result.success is true,
    then output "Agent completed the goal in" agent_result.steps_taken "steps.".
Otherwise,
    output "Agent stopped:" agent_result.reason.

Human-in-the-Loop

hitl.fl Fluent 
-- Pause and require human approval before irreversible actions
@tools(web_search, send_email)
@agent(model: anthropic/claude-sonnet-4-6)
@require_approval_before(send_email)   -- human confirms before any email is sent

Run agent with goal "Draft and send outreach emails to the 5 highest-value leads in leads.csv"
    and call the outcome result.

-- Multi-agent: one agent orchestrates others
@agent(model: openai/gpt-4o, role: orchestrator)
Define agent planner with goal "Break the project into tasks and delegate to specialist agents".

@agent(model: deepseek/deepseek-coder, role: specialist)
Define agent coder with goal "Implement the code tasks delegated by the planner".

@agent(model: anthropic/claude-opus-4, role: specialist)
Define agent writer with goal "Write all documentation tasks delegated by the planner".

Run multi-agent team planner, coder, writer
    on project "Build a REST API for the Saviee app with full documentation"
    and call the outcome project_result.

Reflection & Self-Correction

reflection.fl Fluent 
-- Generate, critique, and refine — classic self-reflection loop
Let draft be empty.
Let quality_score be 0.
Let iterations be 0.

While quality_score is less than 0.85 and iterations is less than 5:
    -- Generate (or improve the previous draft)
    Ask claude to "write a persuasive product description for Saviee app"
        using previous draft draft
        and call the result draft.

    -- Critique with a different model
    @model(temperature: 0, format: json)
    Ask gpt to "score this product description on clarity, persuasion, and accuracy from 0 to 1"
        using text draft
        and call the result critique.

    Set quality_score to critique.average_score.
    Set iterations to iterations plus 1.
End while.

Output draft.

12 — Testing

Testing Fluent Programs

AI programs are probabilistic — the same input can yield slightly different outputs. Fluent's test system is designed for this reality: tests can be exact, semantic, or model-judged.

Writing Tests

tests.fl Fluent 
-- Test file: test_summarizer.fl
Use the procedures from "./summarizer.fl".

-- Exact match test
Test "summarize returns non-empty text":
    Let result be the result of summarize an article "The sun is a star." in "English".
    Expect result to not be empty.
    Expect result to have fewer than 100 words.
End test.

-- Semantic test (model-judged)
Test "sentiment is negative for angry review":
    Let result be the result of classify sentiment "This product is absolutely terrible.".
    @judge(model: gpt, criteria: "Does the result indicate negative sentiment?")
    Expect result.sentiment to semantically match "negative or unfavorable".
End test.

-- Regression test with golden output
Test "entity extraction matches golden set":
    Let result be the result of extract entities "Apple was founded by Steve Jobs in 1976.".
    Expect result to contain "Apple", "Steve Jobs", "1976".
End test.

-- Run with: fluent test tests/

Mocking Model Calls

mock.fl Fluent 
-- Mock a model for deterministic, free testing
When testing, mock claude with the response "Mocked summary: This article discusses AI."
    whenever the instruction contains "summarize".

-- Replay from a fixture file
When testing, replay claude responses from "./fixtures/claude_responses.json".

14 — Runtime & Execution Model

The Fluent Runtime

The Fluent runtime is a three-layer system: a natural language parser that produces an Abstract Semantic Graph (ASG), a planner that resolves model bindings and schedules parallel execution, and an executor that manages async I/O, retries, and result coercion.

Layer 1 — Parser

Converts .fl source into an Abstract Semantic Graph. Uses a grammar model trained on English patterns + Fluent syntax. Reports ambiguous sentences as parse errors.

Layer 2 — Planner

Resolves model aliases, detects independent branches for parallel execution, validates tool availability, and compiles the ASG into an execution plan.

Layer 3 — Executor

Runs the plan. Manages async API calls, exponential retry, credential injection, result type coercion, streaming, and cost tracking.

Observability

Every execution emits structured traces: model called, tokens used, latency, cost, and result hash — to stdout, file, or any OpenTelemetry endpoint.

CLI Reference

terminal Shell 
# Install
npm install -g fluent-lang          # Node runtime
pip install fluent-runtime          # Python binding

# Run a program

# Run with environment

-- AI-powered personal finance advisor with memory and personalization

        and call the result user_memory.

    @model(temperature: 0, format: json)
    Ask groq/llama-4-scout-instant

        to "identify unusual spending patterns or potential savings opportunities"
        using data categorized
        and call the result insights.

    Ask finance_advisor
        to "give personalized advice on how to better meet their financial goals this month"
        using data a record with
            spending categorized,
            insights insights,
            goals user_memory
        and call the result advice.

    -- Remember what was advised
    Remember a record with user user_id, advice advice, date the current date
        as "monthly advice session for user" user_id.

    Return advice.
End of advise user on their finances.
