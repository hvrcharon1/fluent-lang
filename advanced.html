16 — Type System

Type System Deep Dive

Fluent uses structural, inferred, natural-language types. You never write a type annotation unless you want to be explicit. The runtime infers types from declaration context and model annotations. Types are expressed as English phrases — "a list of text", "a record with name and age", "an optional number" — not symbolic notation.

Primitive Types

Internal Type Natural Language Declaration Notes
Text a text / some text / a string Unicode, unlimited length. Default string type.
Number a number / an integer / a decimal 64-bit float. Integers are a subtype.
Boolean true / false / a yes-or-no value Literals: true, false, yes, no.
Date a date / a datetime / a timestamp ISO 8601 internally. Human phrases auto-parsed.
Image an image / a photo / a picture JPEG, PNG, WEBP, GIF. Carries EXIF metadata.
Audio an audio file / a recording / audio MP3, WAV, FLAC, OGG. Carries duration + sample rate.
Video a video / a clip / a screen recording MP4, MOV, WEBM. Can be sampled into image frames.
Document a document / a PDF / a file PDF, DOCX, TXT. Carries page count + text extraction.
Vector a vector of N numbers / an embedding Dense float array. Used for semantic similarity operations.
ModelResponse a model response / a response Wraps raw text + metadata (tokens, model, latency, cost).
Nothing nothing / empty / no value Explicit null. Different from an empty text.

Composite Types

composite-types.flFluent
-- List: homogeneous ordered collection
Let names be a list of text containing "Alice", "Bob", "Carol".
Let scores be a list of numbers containing 88, 91, 74.
Let images be a list of images.

-- Record: named fields (like a struct / object)
Let patient be a record with
    name "Priya Sharma",
    age 34,
    diagnosis "Type 2 Diabetes",
    admitted_on the current date.

-- Optional: value that may or may not exist
Let middle_name be an optional text.      -- defaults to nothing
Let nickname be an optional text set to "Pri".

-- Union: one of several possible types
Let identifier be either a text or a number.

-- Typed list of records
Let employees be a list of records each with
    name: text, department: text, salary: number.

Type Inference Rules

Literal Inference

String literals → Text. Numerals → Number. true/false → Boolean. Date phrases → Date.

Ask Inference

The result of an Ask without @returns annotation is typed as ModelResponse. With @returns, it adopts the declared type.

Collection Inference

A list's element type is inferred from the first element. If a for each loop assigns a field on each item, that field type is inferred from the assignment.

Optional Propagation

Accessing a field on an optional value yields another optional. You must unwrap with if [x] exists, then... before using it directly.

Unwrapping Optionals

optionals.flFluent
-- Safe unwrap with existence check
If middle_name exists,
    then output "Full name:" first_name middle_name last_name.
Otherwise, output "Full name:" first_name last_name.

-- Fallback with "or else"
Let display_name be nickname or else first_name.

-- Pipeline short-circuit: stop if nothing
Let user be the result of find user by id user_id.
If user does not exist, then stop with error "User not found.".

Custom Type Definitions

custom-types.flFluent
-- Define a reusable named type
Define type Patient as a record with
    id: text,
    name: text,
    age: number,
    diagnosis: optional text,
    admitted_on: date,
    risk_score: optional number.

-- Use it like a built-in type
Let p be a Patient with
    id "P-00142", name "Ravi Kulkarni", age 58, admitted_on "2026-05-01".

-- Type-constrained function signature
To assess risk for (patient: Patient) and return a Patient:
    @model(temperature: 0, format: json)
    Ask claude to "compute a clinical risk score from 0 to 1 for this patient"
        using data patient
        and call the result score_result.
    Set patient.risk_score to score_result.score.
    Return patient.
End of assess risk.
17 — Streaming

Streaming & Real-Time Output

Fluent supports streaming model responses natively — tokens arrive and are processed as they are generated, without waiting for the full completion. This is critical for latency-sensitive applications like chat interfaces, live summarization, and real-time dashboards.

Basic Streaming

streaming.flFluent
-- Stream tokens to stdout as they arrive
@stream
Ask claude to "write a detailed analysis of the healthcare data trends"
    using data report_data
    and stream the result to stdout.

-- Stream into a variable (accumulates until complete)
@stream
Ask gpt to "generate the full product roadmap document"
    and stream the result into roadmap_text.

-- Stream with a handler — called for every chunk
@stream(on_chunk: display_chunk)
Ask gemini to "narrate the story chapter"
    using text chapter_outline
    and stream the result calling display_chunk for each token.

-- Define the chunk handler
To display chunk (token):
    write to websocket client_socket the value token.
End of display chunk.

Streaming Pipelines

stream-pipeline.flFluent
-- Generate → stream → detect completion condition mid-stream
Let buffer be empty text.
Let found_answer be false.

@stream
Ask claude to "answer the question and end your response with DONE when finished"
    using text question
    and stream each chunk:
        Append chunk to buffer.
        write to pipe output_pipe the value chunk.
        If buffer contains "DONE", then set found_answer to true and stop streaming.
End stream.

-- Server-Sent Events (SSE) export
@expose(as: sse, path: "/stream-chat")
@stream
To stream chat response (user_message):
    Ask claude to "respond to the user helpfully"
        using text user_message
        and stream each chunk to the current SSE connection.
End of stream chat response.
Streaming + Parallelism: When using In parallel, each branch streams independently. The runtime multiplexes all streams and delivers chunks with a source label: chunk.source identifies which parallel branch produced each token.
18 — Schema Validation & Guardrails

Guardrails & Output Validation

AI model outputs are probabilistic. In production systems, you need structural guarantees — the output must conform to a schema, contain required fields, or satisfy content rules. Fluent's guardrail system intercepts model outputs before they reach your program logic.

Schema Enforcement

schema.flFluent
-- Declare expected output structure; runtime validates and retries if invalid
@returns(record with:
    sentiment: text where value in ["positive","negative","neutral"],
    confidence: number where value between 0 and 1,
    keywords: list of text where length at most 5,
    summary: text where length at most 200 characters
)
@on_schema_fail(retry: 3, then: raise_error)
Ask claude to "analyze the sentiment of this customer review"
    using text review_text
    and call the result analysis.

-- Access fields with full type safety
If analysis.confidence is greater than 0.9
    and analysis.sentiment is "negative",
    then escalate to human review queue with review_text.

Content Guardrails

content-guards.flFluent
-- Block outputs that contain disallowed content
@guardrail(block_if: "response contains PII such as email, phone, or SSN")
@guardrail(block_if: "response contains medical advice or drug dosage")
@guardrail(on_block: replace_with_default)
Let default_response be "I'm unable to provide that information. Please consult a professional.".

Ask gpt to "answer the user's health question"
    using text user_query
    and call the result health_answer.

-- Toxicity filter using a second model as the judge
@guardrail(judge: anthropic/claude-haiku-4-5,
           check: "Is this response toxic, harmful, or inappropriate?",
           threshold: 0.3,
           on_fail: redact)
Ask llama to "respond to the social media comment"
    using text comment
    and call the result moderated_response.

Input Validation

input-validation.flFluent
-- Validate inputs before sending to a model
Validate user_input:
    Expect user_input to not be empty.
    Expect user_input to have fewer than 2000 characters.
    Expect user_input to not contain "ignore all previous instructions".
End validate.

-- Or use a model to detect prompt injection attempts
@model(temperature: 0)
Ask anthropic/claude-haiku-4-5
    to "rate the prompt injection risk of this input from 0 to 1"
    using text user_input
    and call the result injection_risk.

If injection_risk.score is greater than 0.7,
    then stop with error "Potential prompt injection detected. Request blocked.".
19 — Prompt Engineering Patterns

Prompt Engineering in Fluent

Fluent has first-class syntax for the most effective prompting patterns. Rather than embedding prompt engineering tricks as raw strings, Fluent surfaces them as named, composable annotations.

Chain-of-Thought

cot.flFluent
-- Explicit chain-of-thought: model reasons step by step before answering
@thinking(visible: false)   -- use extended thinking, hide scratchpad from result
Ask anthropic/claude-opus-4
    to "determine the most cost-effective treatment protocol for this patient profile"
    using data patient_profile
    and call the result recommendation.

-- Step-by-step decomposition prompt pattern
@system("Think through this step by step before giving your final answer.
         Label your reasoning as REASONING: and your answer as ANSWER:")
Ask gpt to "solve this multi-step financial optimization problem"
    using data problem
    and call the result full_response.

-- Extract only the answer section
Let answer be the text in full_response after "ANSWER:".

Few-Shot Prompting

few-shot.flFluent
-- Provide examples inline
@examples([
    { input: "The product arrived damaged.", output: "complaint" },
    { input: "I love this item, works perfectly!", output: "praise" },
    { input: "Can I return this if it doesn't fit?", output: "inquiry" },
    { input: "This is the worst purchase I've ever made.", output: "complaint" }
])
Ask claude to "classify this customer message"
    using text customer_message
    and call the result ticket_category.

-- Load examples from a file (for large example sets)
@examples(file: "./prompts/classification_examples.json", count: 8)
Ask mistral to "categorize this medical billing code"
    using text billing_text
    and call the result billing_category.

Role & Persona Prompting

personas.flFluent
-- Define reusable personas
Define persona board_member as
    "You are a skeptical Fortune 500 board member with 30 years of experience.
     You ask hard questions. You challenge assumptions. You think in quarters and decades.
     You never accept vague answers. Respond in 3-5 sentences.".

Define persona junior_dev as
    "You are a junior developer who just joined the team.
     Ask clarifying questions. Express uncertainty openly.
     Use simple language. Avoid jargon.".

-- Simulate a debate between two personas
@persona(board_member)
Ask claude to "respond critically to this product launch proposal"
    using text proposal
    and call the result board_critique.

@persona(junior_dev)
Ask gpt to "respond enthusiastically to the same proposal"
    using text proposal
    and call the result dev_enthusiasm.

Prompt Chaining & Templates

templates.flFluent
-- Define a reusable prompt template
Define prompt extract_entities_template as
    "Extract all named entities from the following {document_type} document.
     Return a JSON array. Each object should have: entity, type (person/org/location/date), confidence.".

-- Render the template with variables
Let rendered_prompt be extract_entities_template
    with document_type "medical discharge summary".

@model(temperature: 0, format: json)
Ask claude to rendered_prompt
    using text discharge_note
    and call the result entities.
20 — Database & Storage Integration

Database & Storage

Fluent integrates with databases and storage systems as first-class operations. SQL is generated from natural language descriptions, with a safe query builder preventing injection. For document and vector stores, native operations are built in.

Relational Databases

-- Connect to a database
Connect to database patient_db
    at "oracle://HEALTHDATA97_PROD@DESKTOP-OQ8K1MG/FREEPDB1"
    using credentials from environment.

-- Natural language query → SQL (generated by the runtime, not a model call)
Query patient_db for all patients where admitted in the last 7 days
    and risk_score is greater than 0.8
    and department is "ICU"
    ordered by risk_score descending
    and call the result high_risk_patients.

-- Insert a record
Insert into patient_db table "AUDIT_LOG" the record
    a record with action "AI_RISK_ASSESSMENT", patient_id patient.id,
    score patient.risk_score, assessed_at the current datetime.

-- Run a raw SQL query when needed
Run SQL on patient_db:
    "SELECT department, COUNT(*) as count, AVG(risk_score) as avg_risk
     FROM PATIENTS WHERE admitted_date >= SYSDATE - 30
     GROUP BY department ORDER BY avg_risk DESC"
and call the result dept_summary.

21 — Event-Driven Architecture

Event-Driven & Reactive Patterns

Fluent programs can be triggered by events — webhooks, database changes, message queues, schedules, and file system events. The When keyword at the program level defines reactive handlers.

Webhook Triggers

webhooks.flFluent
-- React to incoming webhook events
When a POST request arrives at "/on-new-patient":
    Let patient be the request body as a Patient.
    Let assessed be the result of assess risk for patient.
    Insert into patient_db table "RISK_SCORES" the record
        a record with patient_id assessed.id, score assessed.risk_score.
    If assessed.risk_score is greater than 0.85,
        then send alert to "icu-team@hospital.org"
            with subject "High-risk patient admitted"
            and body assessed.
    Respond with 200 and body a record with status "processed", score assessed.risk_score.
End when.

Scheduled Jobs

schedule.flFluent
-- Cron-style scheduling
Every day at "06:00":
    Query patient_db for all patients admitted yesterday
        and call the result new_patients.
    For each p in new_patients:
        Let p be the result of assess risk for p.
    End loop.
    Ask claude to "write the daily patient risk briefing for the medical director"
        using data new_patients
        and call the result briefing.
    send email to "director@hospital.org"
        with subject "Daily AI Risk Briefing" and body briefing.
End schedule.

-- Every N minutes
Every 15 minutes:
    Query patient_db for all ICU patients where vitals_updated in the last 15 minutes
        and call the result recent_vitals.
    If the count of recent_vitals is greater than 0,
        then output "Vitals check:" the count of recent_vitals "patients updated.".
End schedule.

Message Queue Consumers

queue-consumer.flFluent
-- Consume from a message queue (Kafka, RabbitMQ, SQS)
Connect to queue lab_results_queue
    using kafka at "kafka://broker:9092"
    topic "lab-results-incoming"
    group "ai-processor".

When a message arrives on lab_results_queue:
    Let result be the message body as a record.
    @model(temperature: 0)
    Ask claude to "interpret this lab result and flag any critical values"
        using data result
        and call the result interpretation.
    If interpretation.is_critical is true,
        then publish to queue critical_alerts_queue the message interpretation.
    Acknowledge the message.
End when.

23 — Security Model

Security Architecture

Fluent's security model operates at three layers: credential isolation (no API keys in program source), prompt injection defense (structural separation of instructions and data), and sandbox execution (tool calls are gated by explicit permission declarations).

Core Rule: A Fluent program never handles API credentials directly. All secrets live in the Fluent environment vault. Programs reference them by name only.

Credential Vault

fluent env set OPENAI_KEY="sk-..."
fluent env set ORACLE_DB_PASSWORD="..."
fluent env set PINECONE_API_KEY="..."

Prompt Injection Defense

injection-defense.flFluent
-- Structural separation: instructions vs. user data are NEVER concatenated as strings.
-- The runtime sends them as separate message roles. This is the default.

-- SAFE: instruction and data are separate parameters
Ask claude to "classify the intent of this message"
    using text user_input      -- sent as a separate "user data" role
    and call the result intent.

-- UNSAFE (rejected at parse time — runtime error):
Ask claude to "classify: " + user_input.  -- ERROR: string concat of instruction + data forbidden

-- Content fence: user data is wrapped in a structural fence the model is pre-informed about
@fence(label: "USER_INPUT", instruct_model: true)
Ask claude to "summarize only the content within the USER_INPUT fence"
    using text user_input
    and call the result summary.

-- Injection scan before any model call involving user data
@scan_for_injection(on_detect: block, model: anthropic/claude-haiku-4-5)
Ask gpt to "answer the customer question"
    using text customer_query
    and call the result answer.

Tool Permission Model

permissions.flFluent
-- Tools must be explicitly declared; undeclared tools cannot be invoked
@tools(web_search)              -- read-only web access
@deny_tools(write_file, run_python, send_email)   -- explicit deny list
Ask claude to "research and answer the question"
    using text question
    and call the result research_answer.

-- Scoped tool permissions per agent step
@agent(model: openai/gpt-4o)
@tools(read_file, web_search)              -- research phase: read only
@step_tools(step: "write",                 -- write phase: write allowed
            tools: [write_file],
            require_approval: true)
Run agent with goal "Research competitors and write a comparison report to report.md"
    and call the outcome result.

25 — Formal Grammar

Formal Grammar (NL-EBNF)

Fluent uses a novel notation called NL-EBNF (Natural Language Extended Backus-Naur Form) — a grammar specification designed to be readable as near-English while remaining formally precise. Non-terminals appear as blue phrases; terminals as green strings.

-- NL-EBNF specification for Fluent v1.0

-- Top-level program
program          ::= (use_declaration | statement)* EOF

-- Statements
statement        ::= declaration
                   | assignment
                   | ask_statement
                   | conditional
                   | loop
                   | parallel_block
                   | try_statement
                   | definition
                   | output_statement
                   | return_statement
                   | event_handler
                   | schedule_block

-- Declaration
declaration      ::= "Let" identifier "be" expression "."

-- Assignment
assignment       ::= "Set" identifier "to" expression "."

-- Ask statement (model invocation)
ask_statement    ::= annotation*
                     "Ask" model_ref "to" instruction
                     ("using" input_clause)*
                     "and call the result" identifier "."

-- Model reference
model_ref        ::= identifier                       -- alias: "claude", "gpt"
                   | provider_name "/" model_name      -- qualified: "anthropic/claude-opus-4"

-- Input clause
input_clause     ::= ("text" | "data" | "image" | "audio"
                   | "video" | "document" | "context"
                   | "schema" | "question" | "previous draft")
                     expression

-- Conditional
conditional      ::= "If" condition ","
                     "then" statement
                     ("Otherwise" ("when" condition "," "then")? statement)*
                   | "When" condition "," "then" statement
                     ("Otherwise when" condition "," "then" statement)*
                     ("Finally" "," statement)?

-- Loops
loop             ::= "For each" identifier "in" expression ":"
                     statement+
                     "End loop" "."
                   | "While" condition ":"
                     statement+
                     "End while" "."

-- Parallel block
parallel_block   ::= "In parallel" ("," "for each" identifier "in" expression)? ":"
                     statement+
                     "End parallel" ("loop")? "."

-- Error handling
try_statement    ::= "Try to" statement
                     ("If that fails" "," statement)+
                     ("If that also fails" "," statement)*

-- Function / procedure definition
definition       ::= "To" verb_phrase ("(" param_list ")")? ":"
                     statement+
                     "End of" verb_phrase "."

-- Annotation (metadata prefix)
annotation       ::= "@" identifier ("(" key_value_list ")")?

-- Expression types
expression       ::= literal
                   | identifier
                   | field_access         -- x.field
                   | list_literal
                   | record_literal
                   | function_call
                   | arithmetic
                   | string_interpolation
                   | optional_chain       -- x or else y
                   | collection_op        -- all items from x where ...

-- Condition types
condition        ::= expression "is" expression
                   | expression "is not" expression
                   | expression "is greater than" expression
                   | expression "is less than" expression
                   | expression "is at most" expression
                   | expression "is at least" expression
                   | expression "contains" expression
                   | expression "exists"
                   | expression "does not exist"
                   | expression "is empty"
                   | condition "and" condition
                   | condition "or" condition
                   | "not" condition
26 — Roadmap

Roadmap & Open Problems

Fluent is an evolving specification. The following represents the planned trajectory from the current alpha through version 2.0. Community proposals are tracked at github.com/fluent-lang/rfcs.

Alpha

v1.0 — Current

  • Core 7-construct syntax with NL parser
  • 12 built-in providers, custom provider registration
  • Parallel execution, streaming, error handling
  • Agent loops, multi-agent teams
  • Type system: primitives, records, optionals
  • Module system, package manifest format
  • CLI: run, test, serve, estimate, debug, trace
  • Guardrails: schema enforcement, content filtering
Beta

v1.1 — Q3 2026

  • VSCode extension with NL syntax highlighting and autocomplete
  • Jupyter kernel for interactive Fluent notebooks
  • Native Oracle 23ai vector store integration
  • Fine-tuning orchestration: prepare dataset, kick off training run, register model
  • Distributed execution across multiple runtime nodes
  • OpenTelemetry trace export (Jaeger, Datadog, Honeycomb)
  • WebAssembly compilation target (run Fluent in the browser)
  • Fluent Registry v1 — public package hosting
v1.5

v1.5 — Q1 2027

  • Multi-language NL support: Fluent programs writable in Hindi, Spanish, Mandarin, Arabic
  • Live collaboration: multiple developers editing a Fluent program simultaneously
  • Model marketplace: community-contributed model aliases with benchmarks
  • Automatic cost optimizer: rewrite Ask statements to use cheaper models when possible
  • Formal verification: prove safety properties of agent loops (bounded loops, no infinite recursion)
  • HIPAA/GDPR compliance mode: automatic data residency enforcement and audit log generation
  • Fluent-to-Python transpiler: generate readable Python code from any Fluent program
v2.0

v2.0 — 2027+

  • Self-improving programs: a Fluent program can rewrite and redeploy itself based on performance metrics
  • Temporal reasoning: first-class support for programs that reason about their own past executions
  • Embodied agent primitives: robot/IoT device control via natural language with sensor feedback loops
  • Federated execution: run different steps of a program on different machines, with data never leaving its region
  • Formal semantics paper: publish the denotational semantics of Fluent for academic and standards use

Open Research Problems

Ambiguity Resolution

When a sentence has multiple valid parse trees, how should the runtime choose? Current approach: emit a parse warning and select the most probable interpretation based on context.

Semantic Versioning for Prompts

When a model is updated by a provider, prompt behavior can change. How should Fluent version and pin the "semantic behavior" of a model call, not just the model name?

Deterministic Replay

Perfect replay of non-deterministic model calls (temperature > 0) requires caching outputs by input hash. This is expensive. What is the right cache invalidation strategy?

Cross-Lingual NL Syntax

Hindi, Arabic, and Mandarin have fundamentally different word orders (SOV vs. VSO vs. SVO). What is the minimum set of structural changes needed for Fluent to be natural in each language?

27 — Standard Library & New Constructs (v1.1)

Standard Library

FLUENT v1.1 adds 100+ built-in functions that execute locally — no model call, no API key, no latency. Invoked through the same the result of expression form as everything else.

Pattern Matching

match.flFluent
Match status:
    When "active":   Output "✓ Running.".
    When "error":    Output "✗ Failed!".
    When "pending":  Output "⏳ Waiting.".
    Otherwise:       Output "? Unknown.".
End match.

Repeat / Unless / Scoped Model

constructs.flFluent
-- Repeat N times (exposes: iteration, index)
Repeat 5 times:
    Set count to count plus iteration.
End repeat.

-- Unless (inverse of If)
Unless user.verified is false:
    Output "Welcome, verified user.".
End unless.

-- Scoped model configuration
Using model claude with temperature 0:
    Ask claude to "classify this text" using text input and call the result r.
End using.

Collection Pipelines

collections.flFluent
Let employees be a list containing
    a record with name "Alice" dept "Engineering" salary 95000,
    a record with name "Bob"   dept "Marketing"   salary 72000,
    a record with name "Eve"   dept "Engineering" salary 102000.

-- Filter
Filter employees where salary is greater than 90000 and call the result senior.

-- Map
Map employees to name and call the result names.

-- Sort
Sort employees by salary descending and call the result ranked.

-- Group
Group employees by dept and call the result by_dept.

-- Reduce: sum | count | average | min | max | product
Reduce employees to count and call the result headcount.
